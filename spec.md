# アルバイト給与管理アプリ 要件定義書

> 最終更新: 2026-05-28 / フェーズ3完了・フェーズ4着手前

---

## 概要

大学生・主婦層のアルバイト収入を管理するスマートフォンアプリ。掛け持ちバイトに対応し、扶養の壁までの残り枠をリアルタイムで把握できる。2026年以降の税制改正（178万円の壁）にも対応し、税制ニュースの周知機能も持つ。

### ターゲットユーザー
- 大学生（掛け持ちバイト、親の扶養内で働きたい）
- 主婦層（配偶者の扶養内で働きたい）

### 差別化ポイント（vs シフトボード等）
- 「シフト管理」ではなく**「扶養を守るための収入管理」**に特化
- 2026年税制改正（178万円の壁）に即日対応できるリモートJSON設計
- **データは完全ローカル保存**。収入・個人情報をサーバーに送信しない
- 給与明細の実績と見込みのハイブリッド集計で正確な年収把握
- 交通費（非課税）と給与（課税）を分離して管理

## 技術スタック

| レイヤー | 採用技術 |
|---|---|
| フロントエンド | React Native + Expo GO |
| 言語 | TypeScript |
| ルーティング | Expo Router |
| ローカルDB | SQLite（expo-sqlite） |
| 税制ルール管理 | GitHub JSON（起動時fetch） |
| 課金 | RevenueCat（月額200円 / 買い切り500円） |
| 既読管理 | AsyncStorage（税制ニュースの既読フラグのみ） |

### 設計方針
- **データはすべてローカル保存**。ユーザーの収入情報をサーバーで管理しない
- 複数端末同期は対象外
- 税制の数値（壁の金額）はアプリにハードコードせず、GitHubのJSONファイルで管理し、起動時にfetchして更新する
- 既読フラグなど消えても実害のない軽微なデータのみAsyncStorageを許可する

## 課金プラン

「基本機能は使えるが制限でちょっとストレスが残る→それを解消するための課金」設計。

| プラン | 価格 | 仕組み |
|---|---|---|
| 無料 | 0円 | デフォルト。制限あり |
| 月額プレミアム | 200円/月 | RevenueCat（サブスク） |
| 永久ライセンス | 500円（買い切り） | RevenueCat（Non-consumable） |
| 開発者コード | 無料・永久解除 | GitHub JSONのSHA256ハッシュ照合 |

### 無料/有料の制限一覧

| 機能 | 無料 | 有料 | 狙い |
|---|---|---|---|
| バイト先登録数 | **2件まで** | 無制限 | 掛け持ちユーザーへの直接的な制限 |
| シフト表示期間 | **直近3ヶ月のみ** | 全期間 | 過去データを見たくなったタイミングで課金誘導 |
| 給与明細の画像添付 | **なし** | あり | 明細管理を本格的にやりたいユーザーへ |
| 収入予測グラフ | **なし** | あり | 将来の見通しを立てたいユーザーへ |
| ペース警告 | **なし** | あり | 壁を超えそうなタイミングで課金誘導 |

### 無料でも使える機能（制限なし）
- オンボーディング・扶養壁の自動判定
- バイト先登録（2件まで）
- シフト入力・見込み給与計算（直近3ヶ月表示）
- 給与明細入力（画像添付なし）
- ホーム画面（残り枠・ゲージ）
- 月次グラフ（過去）
- 税制ニュース表示
- 確定申告サポート（申告要否判定・e-Tax誘導）← 無料

### 課金誘導のタイミング
- 3件目のバイト先を登録しようとしたとき
- 3ヶ月より前のシフトを見ようとしたとき
- 給与明細に画像を添付しようとしたとき
- 予測グラフ・ペース警告にアクセスしたとき

### 開発者コードの仕組み
- `tax_rules.json` の `promo_codes` 配列にSHA256ハッシュを格納
- アプリ内でコードを入力 → ハッシュ化 → JSONと照合
- 一致したらSQLiteの `user_settings.plan = 'lifetime'` に更新
- GitHubのJSONを更新するだけでコードの追加・無効化が可能（アプリ更新不要）
- **Appleガイドライン上の注意**: 無料で配布する分には問題なし。金銭のやり取りはRevenueCat経由のみ

## 扶養壁の判定ロジック（2026年12月時点のルール）

> **基準年**: 2026年分のみを対象とする。アプリの目的は「今年の収入を壁以内に収める」ことであり、過去年度の遡及計算は対象外。

オンボーディングの回答からロジックで壁を自動判定する。ユーザーに「どの壁を気にしますか？」と聞かない。

### 2026年分の壁一覧

| 壁 | 金額 | 制度 |
|---|---|---|
| 自分の所得税 | 178万円 | 所得税法（2026年12月施行） |
| 親の扶養控除・19〜22歳（特定扶養） | 159万円 | 特定親族特別控除（2026年分） |
| 親の扶養控除・23歳以上（一般） | 136万円 | 扶養控除（2026年分改正） |
| 配偶者控除（満額） | 169万円 | 配偶者控除（2026年分改正） |
| 社保の扶養・19〜22歳 | 150万円 | 健康保険（特定扶養特例） |
| 社保の扶養・一般 | 130万円 | 健康保険 |
| 社保加入義務（従業員51人以上） | 106万円 | 厚生年金（2026年10月撤廃予定） |

> **昼間学生の適用除外**: 106万円の壁（短時間労働者の社会保険加入義務）は「昼間部の学生でないこと」が要件のため、昼間学生には勤務先の規模に関係なくこの壁は存在しない。休学中・夜間・通信制の学生は除外されない（＝106万円の壁が適用される）。

### 判定フロー

```
1. dependent_type == "none"
   → 壁なし（所得税178万円のみ表示・参考値）

2. dependent_type == "parent"
   - 2026年12月31日時点で19〜22歳（特定扶養）
     → 社保の扶養:    150万円  ← primary_wall候補（健康保険の被扶養者認定は150万円未満のまま）
     → 扶養控除:      159万円  ← 特定親族特別控除の満額上限（2026年分）
     → 自分の所得税: 178万円
   - 2026年12月31日時点で23歳以上（一般扶養）
     → 社保の扶養:    130万円  ← primary_wall候補
     → 扶養控除:      136万円
     → 自分の所得税: 178万円

3. dependent_type == "spouse"
   → 社保の扶養:      130万円  ← primary_wall候補
   → 配偶者控除満額: 169万円
   → 自分の所得税:   178万円

4. large_company == true かつ is_student == false かつ 週20時間以上勤務
   → 上記に加えて社保加入義務: 106万円  ← primary_wall候補
   → ※昼間部の学生（is_student == true）は勤務先の規模に関係なく106万円の壁の適用除外。
     したがって学生の場合はこの壁を判定に加えない（休学中・夜間・通信制は学生扱いとせず is_student == false とする）
   → ※2026年10月以降、賃金要件（月8.8万円）は撤廃予定。ただし企業規模要件（従業員51人以上）は
     2027年10月以降に段階的撤廃のため、撤廃後も「週20時間以上 かつ 従業員51人以上の企業」が加入条件として残る
   → ※この106万円の壁（social_insurance_large_company）は、撤廃時に tax_rules.json の
     disabled_walls に追加すればアプリ更新なしで判定から除外できる（後述のJSON構造を参照）

primary_wall = 全壁の中で最小値を採用（ホーム画面のメイン表示）
```

### ユーザー別の実質的なデッドライン

| ユーザー | primary_wall | 理由 |
|---|---|---|
| 19〜22歳・大学生・小企業 | **150万円** | 社保の扶養（150万円）が扶養控除（159万円）より先に来る |
| 23歳以上・大学生・小企業 | **130万円** | 社保が先に来る |
| 主婦・小企業 | **130万円** | 社保が先に来る |
| 従業員51人以上の勤務（週20h以上） | **106万円** | 最初に来る壁 |
| 扶養なし | **178万円** | 所得税のみ（参考表示） |

### GitHub JSON 構造（tax_rules.json）

```json
{
  "version": "2026",
  "updated_at": "2026-07-19",
  "walls": {
    "income_tax": 1780000,
    "dependent_specific_age_min": 19,
    "dependent_specific_age_max": 22,
    "dependent_specific_limit": 1590000,
    "dependent_general": 1360000,
    "spouse_full_deduction": 1690000,
    "social_insurance_specific": 1500000,
    "social_insurance_basic": 1300000,
    "social_insurance_large_company": 1060000,
    "residence_tax_approx": 1100000
  },
  "disabled_walls": [],
  "promo_codes": [
    "sha256ハッシュ文字列"
  ],
  "news": [
    {
      "id": "2026-reform-178",
      "title": "178万円の壁が2026年分から適用",
      "body": "所得税の非課税枠が178万円に拡大されました。ただし社会保険の扶養（130万・150万円）は別制度です。",
      "published_at": "2026-01-01",
      "important": true,
      "target": ["all"]
    }
  ]
}
```

**`disabled_walls`（壁の無効化フラグ）**

- 任意フィールド。`walls` のキー名（例: `"social_insurance_large_company"`）を配列で指定すると、その壁を判定結果から除外できる。
- 2026年10月の106万円の壁（`social_insurance_large_company`）撤廃時に、**アプリを更新せずリモートJSONを差し替えるだけ**で壁を消せるようにするための仕組み。
- 省略した場合や空配列の場合はすべての壁が有効（後方互換）。
- すべての壁が無効化されて空になった場合でも、`calcWalls` は所得税非課税枠を必ず1つフォールバック表示する（primary_wall が未定義になってクラッシュしないため）。

## 確定申告サポートの詳細（無料機能）

- **対象**: 2か所以上からの給与があり、年末調整されない副収入が年20万円超のユーザー
- **提供内容**:
  1. バイト先別・年間収入の集計表示
  2. 「確定申告が必要か」の自動判定
  3. e-Tax入力に必要な数字をコピーしやすい形で表示
  4. e-Taxアプリ（国税庁）へのディープリンク誘導
- **提供しないもの**: API連携による自動申告、PDF書類の生成

## 注意事項・免責

- 本アプリの計算はあくまで目安です。正確な判定は税理士・社会保険労務士にご相談ください
- 税制は毎年変わります。アプリ内の情報は最新でない場合があります
- 住民税の非課税ラインは自治体によって異なります（目安: 110〜120万円）
- 本アプリの税制数値は2026年7月19日時点の法令に基づき検証済みです
